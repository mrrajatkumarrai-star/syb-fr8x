1. Firebase Firestore (Cloud Database)

Store all ERP data on Firebase Firestore for multi-device, real-time access.

Firebase Firestore provides real-time cloud sync. Data is still backed up to localStorage so the app works offline. The free Spark plan allows 1 GB storage and 50,000 reads/day — plenty for most freight offices.
1

Create a Firebase Project

Go to console.firebase.google.com, click Add project, enter a project name (e.g. fr8x-erp), disable Google Analytics if not needed, then click Create project.

2

Register a Web App

In the project overview, click the </> (Web) icon → register app with a name → click Register app. You will see a firebaseConfig object like this:

{ apiKey: "AIzaSy...", authDomain: "fr8x-erp.firebaseapp.com", projectId: "fr8x-erp", storageBucket: "fr8x-erp.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" }

Copy this entire block.

3

Enable Firestore Database

In the left sidebar → Build → Firestore Database → Create database. Choose Start in test mode (allows open read/write for 30 days), then select your region (e.g. asia-south1 for India).

4

Configure Firestore Security Rules

Go to Firestore → Rules tab and set the following (replace with auth rules after testing):

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if true; } } }
For production use, replace with authenticated rules once you integrate Firebase Auth.
5

Paste Config in fr8x Settings

Open fr8x → Settings → Database & Storage. In the Firebase Config (JSON) textarea, paste your config as a valid JSON string. Then click Connect to Firebase. You will see a ✓ Connected status.

2. Gmail via EmailJS

Send emails directly from fr8x using your Gmail account via EmailJS.

1

Sign Up on EmailJS

Go to emailjs.com → Create a free account. The free plan allows 200 emails/month.

2

Add Gmail Service

In EmailJS Dashboard → Email Services → Add New Service → Gmail. Click Connect Account, authorize with your Google account. Save — you'll get a Service ID (e.g. service_abc123).

3

Create an Email Template

Go to Email Templates → Create New Template. Set these fields:

FieldValue
To Email{{to_email}}
Subject{{subject}}
From Name{{sender_name}}
Reply To{{sender_email}}
Message Body{{message}}

Save and copy the Template ID (e.g. template_xyz789).

4

Get Your Public Key

In EmailJS → Account → General → Public Key. Copy it (e.g. abcDEFghiJKL).

5

Enter in fr8x Settings

Go to fr8x → Settings → Email Configuration, select EmailJS (Direct Send), and fill:

  • Sender Name: Your company name
  • Sender Email: Your Gmail address
  • EmailJS Public Key: (from step 4)
  • EmailJS Service ID: (from step 2)
  • EmailJS Template ID: (from step 3)

Click Save Email Settings, then Test Send to confirm.

3. Zoho Mail via EmailJS

Link your Zoho Mail account to fr8x via EmailJS SMTP integration.

1

Create EmailJS Account & SMTP Service

Follow steps 1 and 3–4 from Gmail section above for EmailJS account and template creation.

2

Add Zoho SMTP Service in EmailJS

Dashboard → Email Services → Add New Service → Other (SMTP). Fill in:

SettingValue
SMTP Hostsmtp.zoho.com or smtp.zoho.in
Port465 (SSL) or 587 (TLS)
UsernameYour Zoho email address
PasswordYour Zoho password or App Password (if 2FA)
If 2FA is enabled on Zoho: go to Zoho Account → Security → App Passwords → Generate. Use that as the password.

Save and copy your Service ID.

3

Configure fr8x

Same as Gmail step 5 — paste Service ID, Template ID and Public Key into Settings → Email Configuration.

4. Google Drive Storage

Auto-upload shipment documents to Google Drive when files are attached.

1

Create a Google Drive Folder

Open drive.google.com → Create new folder named fr8x-documents (or similar). Open the folder and copy the Folder ID from the URL:

https://drive.google.com/drive/folders/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs

The bold portion is your Folder ID.

2

Get an OAuth Access Token

Visit OAuth 2.0 Playground:

  • In Step 1, find Drive API v3 and select https://www.googleapis.com/auth/drive.file
  • Click Authorize APIs → sign in with your Google account
  • In Step 2, click Exchange authorization code for tokens
  • Copy the Access token (valid for 1 hour)
Access tokens expire in 1 hour. For production, set up a server-side OAuth refresh flow. For small use, re-generate as needed.
3

Configure in fr8x

Go to Settings → Document Storage Configuration, select Google Drive, paste the Access Token and Folder ID, and click Save Storage Settings.

5. Zoho WorkDrive Storage

Upload documents to Zoho WorkDrive for teams already on Zoho ecosystem.

1

Locate Your WorkDrive Folder ID

Log in at workdrive.zoho.com → open your target team folder → copy the resource_id from the URL:

https://workdrive.zoho.com/workdrive/teams/.../folder/abcd1234efgh
2

Generate OAuth Access Token

Go to api-console.zoho.com:

  • Create a Self Client application
  • Generate a token with scope: WorkDrive.files.ALL
  • Copy the Access Token
3

Configure in fr8x

Go to Settings → Document Storage Configuration, select Zoho WorkDrive, paste the Access Token and Folder ID, click Save Storage Settings.

6. Document Uploads

How to attach, manage, and store shipment documents within fr8x.

How to Upload a Document
1

Navigate to Documents in the sidebar. Click Upload Document.

2

Fill the form: select the linked Job Number, enter a Document Type (BL, CRO, Invoice, etc.), optionally add a description.

3

Click Choose File, select your PDF/image. If Google Drive or Zoho WorkDrive is configured, the file uploads to the cloud automatically. Otherwise it is saved as Base64 in local storage.

Supported file types: PDF, JPG, PNG, XLSX, DOCX. Maximum recommended size: 10 MB per file for smooth performance. Large files may slow down localStorage if cloud storage is not configured.

8. Troubleshooting

Common issues and solutions.

Firebase: "Permission Denied"

Firestore rules are blocking access. Go to Firebase Console → Firestore → Rules and temporarily set allow read, write: if true; for testing. See step 4 above for the rule code.

EmailJS: Emails not sending
  • Verify Public Key, Service ID, and Template ID are all correct (no extra spaces)
  • In EmailJS Dashboard → Email Logs, check for error messages
  • If using Gmail, ensure "Less secure app access" is NOT needed — EmailJS uses OAuth
  • Check monthly send quota (200/month on free plan)
Google Drive: Upload fails
  • Access tokens expire in 1 hour — regenerate from OAuth Playground
  • Ensure the scope https://www.googleapis.com/auth/drive.file is selected
  • Verify the Folder ID is correct and the authorized user has access
Data not syncing to Firebase
  • Check browser console (F12) for Firestore connection errors
  • Ensure the Firebase config JSON is valid (use jsonlint.com to validate)
  • Click the Connect to Firebase button again in Settings
  • Data will still be available in localStorage — no data loss occurs
For additional support, check the developer documentation inside the fr8x ERP or contact your system administrator. All configuration is non-destructive — you can always switch back to local storage mode without losing any data.
